## 1. 前置准备

### 1.1 硬件与系统要求

| 项目 | 最低配置 | 推荐配置 |
|------|----------|----------|
| 操作系统 | Windows 10 64位 | Windows 11 64位 |
| 内存 | 4 GB | 8 GB 以上 |
| 硬盘空间 | 2 GB（含 MySQL + 模型文件） | 5 GB 以上 |
| 网络 | 可访问 PyPI 或已有离线包 | 正常宽带连接 |

### 1.2 必须提前安装的软件

在开始配置前，请确认以下软件已安装：

#### ① Python 3.9 ~ 3.11（必须）

> 本项目在 **Python 3.11.1** 下开发和测试，强烈推荐使用 3.11.x。

- 下载地址：https://www.python.org/downloads/
- 安装时勾选 **"Add Python to PATH"**（非常重要！不勾选后续命令会报错）
- 验证安装：
  ```
  python --version
  # 应输出：Python 3.11.x
  pip --version
  # 应输出：pip xx.x.x
  ```

#### ② MySQL 8.0（必须）

> 推荐使用 **MySQL Installer** 或 **phpStudy** 集成环境（phpStudy 更简单，适合答辩环境）。

**MySQL 官方安装包（推荐）**

- 下载：https://dev.mysql.com/downloads/installer/
- 选择 "MySQL Server 8.0.x" 安装
- 安装时设置 root 用户密码，**请记住这个密码**，后续配置需要用到

**方式B：phpStudy（零基础推荐）**
- 下载：https://www.xp.cn/
- 安装后直接启动 MySQL 5.7 / 8.0
- 默认 root 密码为 `root`

- 验证 MySQL 是否运行（在命令行中执行）：
  ```
  mysql -u root -p
  # 输入密码后看到 mysql> 提示符则说明正常
  ```

### 1.3 项目文件结构确认

拿到源代码后，目录结构应如下所示（所有命令在 `health_web/` 目录下运行）：

```
health_web/                        ← 主项目目录
├── app/                          ← Flask 应用包
│   ├── ml/                       ← 机器学习预测模块
│   ├── models/                   ← 数据库模型
│   ├── routes/                   ← 路由蓝图
│   ├── static/                   ← 静态资源（CSS/JS/图片）
│   └── templates/                ← HTML 模板
├── config.py                     ← 配置文件（从环境变量读取密码）
├── run.py                        ← 启动入口
├── seed_demo.py                  ← 数据库初始化脚本
├── requirements.txt              ← Python 依赖清单
```

> **重要**：`../analysis_output/best_model_tuned.joblib` 和 `../analysis_output/scaler.joblib` 必须存在，否则肥胖风险预测功能无法使用。

## 2. 数据库配置与初始化

### 2.1 启动 MySQL 服务

**Windows（MySQL Installer 安装）**：
```
# 在"服务"中找到 MySQL80，右键启动
# 或在命令行（管理员权限）执行：
net start MySQL80
```

**phpStudy 用户**：直接在 phpStudy 面板点击"MySQL 启动"按钮。

### 2.2 创建数据库

> `seed_demo.py` 会自动创建数据库，但手动创建更保险。推荐两步都做。

打开命令行，执行：
```bash
mysql -u root -p
```
输入密码后，在 `mysql>` 提示符下执行：
```sql
CREATE DATABASE IF NOT EXISTS health_db
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;

-- 验证创建成功
SHOW DATABASES LIKE 'health_db';

-- 退出
EXIT;
```

看到 `health_db` 出现在列表中则说明数据库创建成功。

> **此步骤必须在完成第 3、4 节之后执行**（需要先安装依赖和配置环境变量）。

跳转到第 3、4 节完成后，回到这里执行：

```bash

# 运行初始化脚本
```

## 3. Python 环境与依赖安装

### 3.1 （推荐）创建虚拟环境

虚拟环境可以避免不同项目的依赖冲突，强烈推荐使用。

```bash
# 进入 health_web 目录
cd health_web

# 创建虚拟环境（目录名 venv，可自定义）
python -m venv venv

# 激活虚拟环境
# Windows CMD：
venv\Scripts\activate
# Windows PowerShell：
.\venv\Scripts\Activate.ps1
# Mac/Linux：
source venv/bin/activate

# 激活成功后，命令行前缀会显示 (venv)
```

> **注意**：PowerShell 首次运行可能报"不允许执行脚本"错误，执行以下命令解除限制：
> ```
> Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
> ```

### 3.2 安装 Python 依赖

```bash
# 确保在 health_web 目录且虚拟环境已激活
pip install -r requirements.txt
```

安装的主要包（`requirements.txt` 完整清单）：

| 包名 | 版本 | 用途 |
|------|------|------|
| Flask | 3.0.0 | Web 框架 |
| Flask-SQLAlchemy | 3.1.1 | ORM 数据库操作 |
| Flask-Login | 0.6.3 | 用户登录认证 |
| Flask-Bcrypt | 1.0.1 | 密码加密 |
| Flask-Limiter | 3.5.0 | 接口频率限制（防暴力破解） |
| Flask-Migrate | 4.0.5 | 数据库迁移 |
| PyMySQL | 1.1.0 | MySQL 驱动 |
| python-dotenv | 1.0.0 | 读取 .env 文件 |
| joblib | 1.3.2 | 加载机器学习模型 |
| scikit-learn | 1.3.2 | ML 预处理（StandardScaler） |
| numpy | 1.26.2 | 数值计算 |
| cryptography | 41.0.7 | 加密库 |

安装完成后验证：
```bash
python -c "import flask; import pymysql; import joblib; print('OK')"
# 输出 OK 则安装成功
```

## 4. 环境变量配置

> 这是整个配置过程中 **最容易出错** 的步骤，请仔细阅读。

### 4.1 复制模板文件

```bash
# 在 health_web 目录下执行
# Windows CMD / PowerShell：
copy .env.example .env

# Mac/Linux：
cp .env.example .env
```

### 4.2 编辑 `.env` 文件

用任意文本编辑器（记事本、VS Code 均可）打开 `health_web/.env`，按实际情况填写：

```bash
# MySQL 数据库配置
HEALTH_DB_USER=root
HEALTH_DB_PASSWORD=你的MySQL密码          # ← 改成你安装 MySQL 时设置的密码
HEALTH_DB_HOST=localhost
HEALTH_DB_PORT=3306
HEALTH_DB_NAME=health_db

# Session 加密密钥（随机字符串，可用下方命令生成）
HEALTH_SECRET_KEY=在此粘贴你的随机密钥     # ← 改成随机字符串
```

**生成随机密钥**（在命令行中执行）：

```bash
python -c "import secrets; print(secrets.token_hex(32))"
# 输出类似：a3f8c2e1d4b5a6f7e8c9d0e1f2a3b4c5d6e7f8a9b0c1d2e3f4a5b6c7d8e9f0a1
```
将输出的字符串复制，粘贴到 `.env` 的 `HEALTH_SECRET_KEY=` 后面。

### 4.3 完整 `.env` 示例

```bash
HEALTH_DB_USER=root
HEALTH_DB_PASSWORD=MySQL密码123
HEALTH_DB_HOST=localhost
HEALTH_DB_PORT=3306
HEALTH_DB_NAME=health_db
HEALTH_SECRET_KEY=a3f8c2e1d4b5a6f7e8c9d0e1f2a3b4c5d6e7f8a9b0c1d2e3f4a5b6c7d8e9f0a1
```

> **安全提示**：
> - `.env` 文件已被 `.gitignore` 排除，不会上传到 Git 仓库
> - 不要把真实密码分享给他人
> - 等号两边**不要有空格**，值不需要加引号

### 4.4 验证环境变量

```bash
# 在 health_web 目录下运行
python -c "from config import DevelopmentConfig; c = DevelopmentConfig(); print('DB:', c.MYSQL_HOST); print('OK')"
# 如果没有报错且输出 DB: localhost，说明环境变量配置正确
```

## 5. 系统启动与访问

### 5.1 确认模型文件存在

```bash
# 检查必要的模型文件（从 health_web/ 执行）
```
